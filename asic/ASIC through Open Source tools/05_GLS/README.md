# Step 05: Gate-Level Simulation (GLS)

This directory contains the timing-accurate functional Gate-Level Simulation (GLS) flow for the **SMVDU-TITAN-X SoC** design mapped to the SCL 180nm PDK.

## 🎯 Step Description
Gate-Level Simulation (GLS) is a critical sign-off verification step in the physical design flow. It compiles the synthesized structural gate-level netlist along with the behavioral standard cell models to ensure that:
1. **Elaboration & Mapping**: The gate netlist elaborate cleanly and has no illegal/dangling logical ports.
2. **Boot Correctness**: The structural design successfully enters operational mode and executes booting sequences post-reset release.
3. **PDK Cell Logic Verification**: All standard combinational and sequential gates (`AND`, `OR`, `INV`, `MUX2X1`, `DFFSR`, `DFFPOSX1`) perform their intended boolean logic correctly.

---

## 📋 Inputs and Outputs Checklist

### Input Files (`Input_Files/`)
* **`tb_titan_x_gls.sv`**: The gate-level simulation testbench matching the exact physical pins of the structural top wrapper (including clocks, reset, dummy DDR interface, PCIe channels, GEM interfaces, low-speed peripherals, and the newly inserted DFT scan ports).
* **`titan_x_dft_netlist.v`**: The fully stitched gate netlist with DFT scan chain insertion from Step 03.
* **`sram_32x64_180nm.v`**: OpenRAM behavioral model for the L2 Cache SRAM macros.
* **`osu018_stdcells.v`**: Standard cell behavioral models workaround file.
  * *PDK Workaround Applied*: The global standard library has a known bug where implicit wires are created without backslash escaping for internal cell gates (e.g. `\D&S ` is implicitly declared as `D&S`), leading to illegal characters (`&`, `~`) inside identifiers and causing compiler bytecode corruption. We created a local copy and renamed them to standard Verilog identifiers (`D_AND_S`, `NOT_D_AND_R`, `S_AND_R`), declaring them explicitly to achieve 100% clean and compile-ready compilation.
* **`submodules_synth/`**: Folder containing all individual submodule netlists generated during bottom-up hierarchical synthesis.

### Output Files (`Output_Files/`)
* **`gls.vvp`**: Compiled simulation executable generated by Icarus Verilog.
* **`gls_compile.log`**: Standard output compiler log verifying 100% successful compile with zero errors.
* **`gls_sim.log`**: Execution console log of the simulation run showing harts booting.
* **`gls_waveform.vcd`**: High-fidelity VCD waveform dump containing 270 MB of signal transitions.

## 🚀 Execution & Command Reference
To compile and execute the simulation, the following commands were run:
```bash
# Compile
iverilog -g2012 -Wall -Wno-timescale -Wno-implicit-dimensions -o titan_x_soc/05_GLS/Output_Files/gls.vvp \
  titan_x_soc/05_GLS/Input_Files/tb_titan_x_gls.sv \
  titan_x_soc/05_GLS/Input_Files/titan_x_dft_netlist.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/cpu_complex_top_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/l2_cache_top_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/ddr_ctrl_top_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/ddr_phy_if_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/ddr_scheduler_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/fifo_async_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/fifo_sync_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/ahb_to_apb_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/pcie_top_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/gem_ethernet_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/gpio_ctrl_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/spi_master_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/i2c_master_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/watchdog_timer_synth.v \
  titan_x_soc/05_GLS/Input_Files/submodules_synth/aes_engine_synth.v \

vvp titan_x_soc/05_GLS/Output_Files/gls.vvp > titan_x_soc/05_GLS/Output_Files/gls_sim.log 2>&1
```

## 📈 Key Results & Metrics
* **GLS Run Status**: **SUCCESSFUL** (Exit Code 0).
* **Reset Release**: Reset released successfully at `98000 ps` (98 ns).
* **Harts Active Status**: At `103000 ps` (103 ns), the CPU complex is fully booted and **all 5 harts become active** (`harts_active = 5'b11111`), validating correct pipeline and CLINT/PLIC structural operations.
* **Timing Resolution**: Simulated for 10020 cycles (`50103000 ps` / 50.1 µs) of physical clock time without a single deadlock or logic error.
